Through April 30, 2017, subscribers to the Microsoft Translator Text and Speech APIs have limited access in Azure DataMarket. We recommend that you migrate to Azure now to avoid service disruption.
 
  Here is a summary of changes:
 
• You are not able to upgrade or downgrade your subscription plan.
• DataMarket no longer accepts new subscriptions in 2017.
• Microsoft Translator, part of the Microsoft Cognitive Services collection of APIs, is now available in the Azure portal.

We recommend that you take these steps to migrate:
 
1. Subscribe to Azure if you haven’t done so already.
2. With a minor code update, change existing applications that are using the Microsoft Translator API. For more information on how to do this, visit the Action Required before April 30, 2017: Microsoft Translator Moves to Azure webpage.

Although DataMarket officially retires on March 31, 2017, Microsoft Translator service access will continue through April 30, 2017.

Your Azure account uses a different identity than your Azure DataMarket account. The corrections you have collected using the AddTranslation function will not be available until you migrate them. We offer a simple tool to make your corrections available under your old and your new identity. For more information, please visit the CTF: Migrate your collaborative translations from Azure DataMarket to Azure by April 30, 2017 webpage.

Please plan your move now to the Azure portal to avoid service disruption. For support and additional updates, please visit the Microsoft Translator User Forum for Businesses and Developers.

You will continue to receive these monthly service emails until you cancel your existing Microsoft Translator API subscription in DataMarket.
